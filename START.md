# 🚀 Инструкция по запуску проекта

## Шаг 1: Запустите PostgreSQL

Откройте терминал в папке `server`:
```bash
cd server
docker-compose up -d
```

Проверьте, что контейнер запущен:
```bash
docker-compose ps
```

## Шаг 2: Настройте переменные окружения

### Backend (.env уже создан, но проверьте):
Откройте `server/.env` и убедитесь, что настроены:
- ✅ DATABASE_URL (должно быть как есть для Docker)
- ⚠️ CLOUDINARY_* (нужно получить на cloudinary.com)
- ⚠️ SMTP_* (нужен Gmail App Password)

**Временно можно оставить как есть** - проект запустится, но загрузка изображений и email не будут работать.

### Frontend (.env уже создан):
Файл `client/.env` уже настроен для работы с локальным API.

## Шаг 3: Установите зависимости Backend

```bash
cd server
npm install
```

Ожидайте ~1-2 минуты.

## Шаг 4: Запустите миграции Prisma

```bash
npm run prisma:migrate
```

Эта команда создаст все таблицы в базе данных.

## Шаг 5: Заполните базу тестовыми данными

```bash
npm run prisma:seed
```

Будут созданы:
- 3 тестовых пользователя (admin, ivan, maria)
- Несколько постов
- 2 курса с уроками
- Тестовые заказы

## Шаг 6: Запустите Backend сервер

```bash
npm run dev
```

Вы должны увидеть:
```
🚀 Сервер запущен на порту 5000
✅ База данных подключена
```

**НЕ ЗАКРЫВАЙТЕ** этот терминал!

## Шаг 7: Запустите Frontend (в новом терминале)

Откройте **НОВЫЙ** терминал в папке `client`:
```bash
cd client
npm install
```

Затем запустите:
```bash
npm run dev
```

Вы должны увидеть:
```
  VITE ready in XXX ms
  ➜  Local:   http://localhost:5173/
```

## Шаг 8: Откройте браузер

Откройте `http://localhost:5173`

## 🐛 Проблемы?

### Порт 5000 занят:
В `server/.env` измените `PORT=5000` на `PORT=5001`
И в `client/.env` измените `VITE_API_URL=http://localhost:5001/api`

### База данных не подключается:
```bash
cd server
docker-compose down
docker-compose up -d
npm run prisma:migrate
```

### Ошибки Prisma:
```bash
cd server
npm run prisma:generate
```

### Всё сломалось:
```bash
# Остановите всё
cd server
docker-compose down

# Удалите node_modules
rm -rf node_modules
cd ../client
rm -rf node_modules

# Начните заново с Шага 1
```
